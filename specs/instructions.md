## instructions for Git
- Just create a version when you push
- When pushs, create a version tag, update manifest.json and version.js with new version tag
- The tag should be in the format 1.0.0

## update specs and changelogs
- When push, update CHANGELOG.md with new version and major changes. Write in a way you can have a clear view of what and why was changed in a way you can have context in future interactions
- Update the prd.md and features-info.md with new features and changes
